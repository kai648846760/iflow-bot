Released: 2026-03-18

## Highlights / 版本亮点

- Bound gateway stdio ACP prewarm with an explicit timeout to keep startup responsive even when adapter warmup stalls.
- 为 gateway 的 stdio ACP 预热增加显式超时，避免适配器预热卡住时拖死启动流程。

- Tightened Ralph static-frontend runtime guards so dynamic checkbox creation and localStorage persistence stories are accepted correctly.
- 收紧 Ralph 静态前端运行时守卫，正确接受动态 checkbox 创建和 localStorage 持久化场景。

- Improved canonical todo PRD normalization and safer output directory handling for fresh Ralph runs.
- 改进 canonical todo PRD 规范化逻辑，并为 fresh Ralph run 增加更安全的输出目录处理。

- Expanded regression coverage around recovery paths, startup ordering, stdio timeout retries, PRD delivery, and runtime guards.
- 扩展恢复路径、启动顺序、stdio 超时重试、PRD 交付与运行时守卫相关的回归覆盖。

- Verified the release against a fresh Feishu `/new` run that completed the full Ralph loop, recovered from watchdog cancellations, and survived 500+ real messages plus session compression.
- 已通过一轮从 Feishu `/new` 开始的真实全量验证：完整跑通 Ralph loop，验证 watchdog 取消后的自动恢复，以及 500+ 真实消息与会话压缩场景。
